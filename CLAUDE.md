# CLAUDE.md

このファイルは、このリポジトリで作業するClaude Code (claude.ai/code) にガイダンスを提供します。
Codexで作業する場合は `AGENTS.md` も参照してください。

## リポジトリの目的

このリポジトリは、日本語でCodex用およびClaude Code用のカスタムSkillを開発・管理するためのものです。

- Claude Code向けSkillは `.claude/skills/` に配置します。
- Codex向けSkillは `.codex/skills/` に配置します。
- 可能な限り両者で同じSkill名・同じ目的・同じ利用条件になるように整備します。

## 言語

すべてのコミュニケーション、ドキュメント、Skillの説明文は日本語で記述します。
コード内の識別子、ファイル名、ディレクトリ名は原則として英語またはASCIIのkebab-caseを使用します。

## Skillのファイル構成

各Skillは、対象ツールごとに以下の構造で配置します：

```
.claude/skills/<skill名>/
  SKILL.md        -- Skillの定義・指示（必須）
  scripts/        -- 実行可能なスクリプト（任意）
  templates/      -- テンプレートファイル（任意）

.codex/skills/<skill名>/
  SKILL.md        -- Skillの定義・指示（必須）
  scripts/        -- 実行可能なスクリプト（任意）
  templates/      -- テンプレートファイル（任意）
```

## SKILL.md の書き方

SKILL.md はYAMLフロントマター + Markdown本文の形式で記述します：

```yaml
---
name: <skill名>
description: <一行概要（日本語）>
---
```

本文には、そのSkillが呼び出されたときにエージェントが従うべき指示を自然言語で記述します。
指示は具体的かつ実用的に書き、「〜してください」「〜すること」の形式で統一します。
対象ツール固有の制約がある場合は、`Claude Code向け`、`Codex向け` の見出しを分けて明記します。

## Skill開発の注意点

- Skillは独立して動作すること。他のSkillへの暗黙的な依存を持たせない。
- 外部APIやネットワークが必要な場合はSKILL.mdに明記する。
- 破壊的な操作（ファイル削除、gitの強制操作など）を行うSkillは確認を求める設計にする。
- 秘密情報、APIキー、個人情報、認証済みセッションの内容をSkill本文やテンプレートに含めない。
- サンプルはそのまま実行しても安全な入力に限定する。
- Codex版とClaude Code版を両方作る場合、内容差分は意図が分かるようにコメントまたは本文で説明する。

## 作業時の確認

新しいSkillや既存Skillの変更後は、最低限次を確認します：

- `SKILL.md` に `name` と `description` のフロントマターがある。
- 呼び出し条件、入力、出力、禁止事項が本文から分かる。
- 任意の `scripts/` は実行方法、前提、失敗時の扱いが分かる。
- 日本語として読めるが、ディレクトリ名やコマンド名は機械処理しやすい。
