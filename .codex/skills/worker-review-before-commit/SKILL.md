# Worker Review Before Commit

AI Workerが出したローカル差分をコミットする直前にレビューするためのSkillです。コミット単位として焦点が合っているか、無関係差分や不要生成物が混ざっていないか、最低限の検証証跡がそろっているかを確認し、コミットしてよい状態かを厳しめに判定します。

## 使うべき状況

- AI Workerの差分をローカルで確認し、コミットしてよいか最終チェックしたい。
- 1コミットとしてまとまりがあるか、無関係差分が混ざっていないかを見たい。
- コミット前に最低限の検証がそろっているかを確認したい。

## 期待する入力

- ユーザー依頼または元タスクの目的。
- 対象リポジトリまたは対象ディレクトリ。
- Workerが出した差分の範囲。
- 実行した検証コマンドと結果。
- レビュー対象の時点。例: コミット前。
- 必要なら関連Issue、PR、作業票、README、手順書。

## 実行手順

1. 元タスクの目的と、このコミットが何を含むべきかを確認します。
2. 対象repoの差分を読み、AI Worker由来の変更範囲と無関係差分を切り分けます。
3. コミット単位として焦点が合っているか、不要な整形、生成物、デバッグ痕跡、設定変更が混ざっていないかを確認します。
4. 実行した検証コマンドと結果を見て、このコミットに必要な最低限の検証がそろっているかを確認します。
5. 仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報混入リスク、コミット前として早すぎる変更を重大度順に整理します。
6. 入力不足があれば README、AGENTS.md、関連Skill、差分、テストから補完します。危険な推測になる場合だけ確認し、証跡不足なら `No-Go` にします。
7. このSkillでは push 可否や deploy 可否まで抱え込まず、コミット直前の品質と範囲に集中して判定します。

## 出力形式

- 対象とレビュー時点。
- Findings。
- Open questions または不足証跡。
- 判定。
  - `Go`
  - `No-Go`

Findings は重大度順に並べ、少なくとも仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報や認証情報の混入リスク、時点に対して早すぎる変更を含めます。

## 禁止事項と確認が必要な操作

- このSkill自体は修正、コミット、push、deploy を実行しません。
- レビュー対象が曖昧なまま推測で可否を出しません。
- 認証情報、APIキー、個人情報、認証済みセッション内容を出力しません。
- 本番操作、破壊的操作、外部公開設定変更を自動で進めません。
- ネットワークや外部CI確認が必要でも、証跡不足なら `No-Go` を返して止めます。
- push 前提の共有品質や deploy 固有確認を、このSkillだけで満たしたと見なしません。
