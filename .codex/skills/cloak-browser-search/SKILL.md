# Cloak Browser Search

Web検索や外部サイト確認をするとき、利用可能ならCloakブラウザを最優先で使うためのSkillです。

## 使うべき状況

- ユーザーがWeb検索、ブラウザ検索、外部サイト確認、最新情報確認を依頼した。
- 「調べて」「検索して」「見てきて」「公式サイトを確認して」など、外部Web確認が必要な依頼を受けた。

## 基本方針

- Cloakブラウザを操作できるツール、MCP、コネクタ、またはローカル連携が利用可能なら、通常のWeb検索ツールより先に使います。
- Cloakブラウザで検索できない、またはCloak連携が見つからない場合は、通常検索へ進む前にその理由を短くユーザーへ伝えます。
- 情報源の品質判断は別Skillや既存方針に従います。技術調査では `source-policy` と併用し、一次情報・公式情報を優先します。

## 実行手順

1. 利用可能なツール一覧から、Cloakブラウザを直接操作できる手段を探します。
2. Cloakブラウザが使える場合は、Cloak上で検索または対象URLを開いて確認します。
3. Cloakブラウザが使えない場合は、通常検索に進む前に「Cloak連携がこの環境では見つからない」など、制約を明示します。
4. 検索結果を使う場合は、参照したURLと確認内容を簡潔に報告します。

## 禁止事項

- Cloakブラウザが使えるのに、黙って通常のWeb検索だけで済ませない。
- CloakブラウザをOSレベルの強制機構やセキュリティ境界のように説明しない。
- 認証済みセッション、個人情報、秘密情報を勝手に開いたり引用したりしない。
