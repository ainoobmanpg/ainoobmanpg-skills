# Worker Review Before Push

AI Worker由来のコミット済み変更を push する直前にレビューするためのSkillです。共有してよい品質か、差分説明と検証結果が他者レビュー可能な形でそろっているか、秘密情報や事故要因がないかを確認し、push してよいかを厳しめに判定します。

## 使うべき状況

- AI Workerの変更をコミット済みで、共有先へ push する前に最終確認したい。
- 他者が差分と検証結果を追える状態かを確認したい。
- 秘密情報、履歴事故、説明不足のまま共有しないようにしたい。

## 期待する入力

- ユーザー依頼または元タスクの目的。
- 対象リポジトリまたは対象ディレクトリ。
- Workerが出した差分の範囲。
- 実行した検証コマンドと結果。
- レビュー対象の時点。例: プッシュ前。
- 必要なら関連Issue、PR、作業票、README、手順書。

## 実行手順

1. 元タスクの目的と、共有先へ出すべき変更範囲を確認します。
2. コミット済み差分と履歴を見て、AI Worker由来の変更が説明可能な単位でまとまっているかを確認します。
3. 差分説明、コミットメッセージ、検証結果、関連Issueや作業票への結び付きがあり、他者レビュー可能な証跡になっているかを確認します。
4. 無関係差分、不要生成物、秘密情報、認証情報、事故要因になり得る設定変更が含まれていないかを確認します。
5. 仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報混入リスク、push 前として早すぎる変更を重大度順に整理します。
6. 入力不足があれば README、AGENTS.md、関連Skill、差分、テストから補完します。危険な推測になる場合だけ確認し、証跡不足なら `No-Go` にします。
7. このSkillでは deploy 前提のロールバック確認や本番環境差分確認まで抱え込まず、push 直前に共有してよい品質かへ集中します。

## 出力形式

- 対象とレビュー時点。
- Findings。
- Open questions または不足証跡。
- 判定。
  - `Go`
  - `No-Go`

Findings は重大度順に並べ、少なくとも仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報や認証情報の混入リスク、時点に対して早すぎる変更を含めます。

## 禁止事項と確認が必要な操作

- このSkill自体は修正、コミット、push、deploy を実行しません。
- レビュー対象が曖昧なまま推測で可否を出しません。
- 認証情報、APIキー、個人情報、認証済みセッション内容を出力しません。
- 本番操作、破壊的操作、外部公開設定変更を自動で進めません。
- ネットワークや外部CI確認が必要でも、証跡不足なら `No-Go` を返して止めます。
- deploy 固有の環境差分、ロールバック、外部依存確認を、このSkillだけで代替しません。
