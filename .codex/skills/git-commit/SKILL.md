# Git Commit

ローカル差分を確認し、コミットに含める範囲を絞ってステージングし、日本語のコミットメッセージで `git commit` まで実行するためのSkillです。push、PR作成、deploy、履歴改変は扱いません。

## 使うべき状況

- ローカル差分を1コミットにまとめたい。
- コミット前に、含める変更と除外する変更を確認したい。
- 無関係差分、生成物、デバッグ痕跡、秘密情報を混ぜずにコミットしたい。
- 日本語の件名と本文でコミットメッセージを作りたい。

## 期待する入力

不足があれば `git status`、差分、README、AGENTS.md、関連Issueや作業票から補完します。危険な推測になる場合だけユーザーに確認します。

- 作業目的。
- コミットに含める変更範囲。
- 除外すべき差分、生成物、一時ファイル、未完了作業。
- 実行済みの検証コマンドと結果。
- 関連Issue、PR、作業票、レビュー結果。
- コミットメッセージに入れたい観点や表現。

## 実行手順

1. `git status --short --branch` で現在ブランチ、変更ファイル、未追跡ファイルを確認します。
2. ユーザーの目的と差分を照合し、今回のコミットに含める範囲を決めます。
3. `git diff`、必要に応じて `git diff --stat`、`git diff --cached`、対象ファイルの内容を確認します。
4. 秘密情報、APIキー、個人情報、認証済みセッション内容、不要な生成物、キャッシュ、ログ、デバッグ痕跡が混ざっていないか確認します。
5. 無関係差分や未完了作業があれば、ステージング対象から除外します。除外理由は最終報告に残します。
6. 必要最小限のファイルだけを `git add <path>` でステージングします。対象が曖昧なまま `git add .` や全量ステージングを使いません。
7. `git diff --cached` でコミット内容を最終確認します。
8. 検証結果が未実行または不足している場合は、実行できる範囲で検証します。実行できない場合は理由を記録します。
9. 日本語の件名と本文を作ります。件名は変更内容を簡潔に表し、本文には必要に応じて目的、主な変更、検証結果、関連Issueを書きます。
10. `git commit` を実行します。
11. コミット後に `git status --short --branch` と `git log -1 --oneline` を確認します。

## コミットメッセージ方針

- 人が読む文言は日本語を基本にします。
- 件名は1行で、何を追加、修正、削除したかを具体的に書きます。
- 本文を書く場合は、変更理由、含めた範囲、検証結果を短く整理します。
- 関連Issueがある場合は、本文に `Refs #<番号>` や `Closes #<番号>` を入れるかどうかをユーザー指示とリポジトリ慣例に合わせます。
- 秘密情報、個人情報、認証済みセッション内容、内部トークンをメッセージに含めません。

## 出力形式

最終報告には次を含めます。

- コミットhash。
- コミット件名。
- コミットに含めた範囲。
- 除外した差分があれば、その範囲と理由。
- 実行した検証コマンドと結果。
- コミット後の `git status` 状態。

## `worker-review-before-commit` との違い

- `worker-review-before-commit` は、AI Worker差分をレビューして `Go` / `No-Go` を返すだけのレビュー専用Skillです。修正、ステージング、コミット、push、deploy は実行しません。
- `git-commit` は、ユーザーがコミット実行を求めたときに使います。差分確認、必要最小限のステージング、コミットメッセージ作成、`git commit`、事後確認まで行います。
- AI Worker由来の差分に品質懸念がある場合は、先に `worker-review-before-commit` でレビューし、`Go` 相当の状態になってからこのSkillでコミットします。

## 禁止事項と確認が必要な操作

- このSkillでは push、PR作成、merge、deploy、本番操作を実行しません。
- `git reset --hard`、`git clean`、rebase、amend、force push などの履歴改変や破壊的操作を、ユーザーの明示なしに実行しません。
- コミット範囲が曖昧なまま全量ステージングしません。
- 無関係差分、未完了作業、生成物、キャッシュ、ログ、デバッグ痕跡を混ぜません。
- 秘密情報、APIキー、個人情報、認証済みセッション内容を表示またはコミットしません。
- 外部認証、ネットワーク、GitHub上のIssue更新やCloseが必要な作業は、コミットとは別操作として扱います。
