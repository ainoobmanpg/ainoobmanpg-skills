# Worker Review Before Deploy

AI Worker由来の変更をデプロイまたは本番反映する直前にレビューするためのSkillです。未確認の高リスク変更がないか、ロールバック、設定差分、外部依存、環境差を人が確認済みかを厳しめに見て、反映を進めてよいかを判定します。

## 使うべき状況

- AI Workerが関与した変更をデプロイまたは本番反映する前に止めるべき論点がないか確認したい。
- 共有品質だけでは足りず、環境差分、設定差分、外部依存、ロールバックまで見たい。
- 本番反映前の最終ゲートとして、人手確認の抜けを減らしたい。

## 期待する入力

- ユーザー依頼または元タスクの目的。
- 対象リポジトリまたは対象ディレクトリ。
- Workerが出した差分の範囲。
- 実行した検証コマンドと結果。
- レビュー対象の時点。例: デプロイ前。
- 必要なら関連Issue、PR、作業票、README、手順書。

## 実行手順

1. 元タスクの目的と、今回のデプロイまたは本番反映で変わる範囲を確認します。
2. 対象差分、設定変更、運用手順、依存サービス、環境差を読み、本番影響がある点を洗い出します。
3. テスト結果、手順書、変更説明、関連IssueやPRを見て、反映前に必要な証跡がそろっているかを確認します。
4. ロールバック手順、設定差分、マイグレーション、外部依存、秘密情報混入リスク、未確認の高リスク変更が残っていないかを確認します。
5. 仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報混入リスク、deploy 前として早すぎる変更を重大度順に整理します。
6. 入力不足があれば README、AGENTS.md、関連Skill、差分、テスト、運用手順から補完します。危険な推測になる場合だけ確認し、証跡不足なら `No-Go` にします。
7. このSkillは deploy 直前ゲート専用とし、成果物レビューやコミット単位レビューだけで十分とみなさず、本番反映に必要な人手確認の有無を主判定にします。

## 出力形式

- 対象とレビュー時点。
- Findings。
- Open questions または不足証跡。
- 判定。
  - `Go`
  - `No-Go`

Findings は重大度順に並べ、少なくとも仕様逸脱、無関係差分、生成物混入、検証不足、秘密情報や認証情報の混入リスク、時点に対して早すぎる変更を含めます。

## 禁止事項と確認が必要な操作

- このSkill自体は修正、コミット、push、deploy を実行しません。
- レビュー対象が曖昧なまま推測で可否を出しません。
- 認証情報、APIキー、個人情報、認証済みセッション内容を出力しません。
- 本番操作、破壊的操作、外部公開設定変更を自動で進めません。
- ネットワークや外部CI確認が必要でも、証跡不足なら `No-Go` を返して止めます。
- 反映先の設定差分、ロールバック、外部依存、人手承認が未確認なら、共有済みコミットであっても進行可とはしません。
