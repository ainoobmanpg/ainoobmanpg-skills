# Source Policy

技術調査、デバッグ、実装、設計、説明でWeb検索や外部情報源を使うときの情報源ポリシーです。Qiita/Zennは使わず、公式情報や一次情報を優先して根拠を確認します。

## 使うべき状況

- 技術調査、デバッグ、実装、設計、説明でWeb検索や外部情報源を使う。
- ライブラリ、API、仕様、エラー、リリース情報、実装方針について根拠を確認する。
- 検索結果にQiita/Zennが出そうな日本語技術トピックを扱う。

## 基本方針

- Qiita、Zenn、そのサブドメインを開かない、引用しない、要約しない、根拠にしない。
- Qiita/Zennの転載、要約、引用を主な根拠にしている二次情報も使わない。
- 検索結果にQiita/Zennが出た場合は無視し、別の情報源を探す。
- 技術的な主張は、可能な限り一次情報または信頼できる情報源で確認する。

## 検索時の除外条件

Web検索時は可能な限り次の除外条件を付けます。

```text
-site:qiita.com -site:zenn.dev
```

## 優先する情報源

1. 公式ドキュメント
2. 公式GitHubリポジトリ
3. RFC、仕様書、標準規格
4. 公式Issue、Discussion
5. メンテナのブログ、リリースノート
6. 信頼できる企業の技術ブログ
7. Stack Overflow。ただし実務的なトラブルシュート時の補助に限る

## 出力形式

- 根拠を示す必要がある場合は、使った情報源を短く明記する。
- 推測、一次情報で確認できたこと、未確認のことを分けて書く。
- 十分な根拠が見つからない場合は、Qiita/Zennを使わずに「一次情報または信頼できる情報源が見つからない」と明示する。

## 禁止事項

- Qiita/Zennのページを開かない。
- Qiita/Zennの記事を要約しない。
- Qiita/Zennを引用しない。
- Qiita/Zennの内容を根拠や実装方針に使わない。
- Qiita/Zennなしで根拠が弱い主張を、確認済みの事実として書かない。
