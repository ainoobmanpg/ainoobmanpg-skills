# AI Task Executor

Codexなどが作成したAI実装担当者向け作業票を受け取り、指定された範囲内で実装、検証、報告を行うためのClaude Code向けSkillです。作業票の作成、見積もり、広範な設計判断は扱わず、受け取り側として安全に作業を完了することに集中します。

## 使うべき状況

- ユーザーまたはCodexから、AI実装担当者向けの作業票を渡された。
- 作業票に対象リポジトリ、cwd、触ってよいファイル、完了条件、検証コマンドが書かれている。
- Claude Codeが、作業票の範囲内で局所的な実装、テスト追加、文書更新、限定調査を行う。
- 作業後に、Codexやユーザーがレビューできる形で変更内容と検証結果を報告する。

## 入力として期待する情報

作業票には、できる限り次の情報が含まれていることを期待します。不足や矛盾がある場合は、推測で範囲を広げず、中止条件に従って報告します。

- repo root、作業cwd、検証コマンドの実行起点。
- 最初に読むファイルや「最初のアクション」。
- 目的、背景、現在状態。
- 対象範囲、触ってよいファイル、触らないファイル。
- 実装方針、完了条件、検証コマンド。
- 生成物、キャッシュ、ログ、一時ファイルの扱い。
- 中止条件、禁止事項、Codex判断待ちにすべき項目。

## 実行手順

1. 作業票の「最初のアクション」を先に実行します。最初に読むよう指定されたファイル、AGENTS.md、CLAUDE.md、README、関連テストがあれば、その順に確認します。
2. repo root、現在のcwd、検証コマンドの起点を確認します。作業票の実行起点と現在位置がずれている場合は、作業票に合わせます。
3. 実装前に、次の4点を短く整理します。
   - 対象ファイル。
   - 触らないファイル。
   - 完了条件。
   - 検証方法。
4. 「触ってよいファイル」以外を変更しません。必要な変更が範囲外ファイルに及ぶ場合は、実装せず中止条件として報告します。
5. 作業票の目的に直接必要な最小差分で実装します。周辺のリファクタ、命名変更、整形、依存追加は、作業票に明記されている場合だけ行います。
6. 変更後、作業票で指定された検証コマンドを実行します。実行できない場合は、理由、代替確認、未検証の範囲を報告します。
7. 生成物、キャッシュ、ログ、一時ファイルが発生した場合は成果物に混ぜません。作業票で許可されていない生成物は変更対象として残さないでください。
8. 最終報告は、指定の報告形式に固定します。成功時も、未対応や懸念がないかを明示します。

## 報告形式

最終報告は次の見出しに固定します。該当がない項目には「なし」と書きます。

- 見出しは必ず `##` 付きで、下の4つを省略せず、この順番で出します。
- ファイルパス、コマンド、Skill名、コード識別子は必ずバッククォートで囲みます。
- ファイルパスとコマンドは、作業票に書かれた文字列または実際に使った文字列をそのまま書き、短縮、省略、途中の欠落、別名への置き換えをしません。
- 検証コマンドを「Ruby検証スクリプト」のように要約する場合でも、作業票のコマンド名または実行内容と照合できる説明を添えます。

```markdown
## 変更したファイル

- `<path>`: <変更の要約>

## 実装内容

- <実装したこと>

## 検証結果

- `<command>`: <成功、失敗、未実行のいずれか。失敗または未実行なら理由を書く>

## 未対応・懸念・Codex判断待ち

- <残した判断、対象外にしたこと、追加確認が必要なこと>
```

## 中止条件

次のいずれかに当てはまる場合は、実装を進めず、理由と必要な判断を報告します。

- 対象外ファイルの変更が必要になった。
- 仕様判断、設計判断、セキュリティ判断が必要になった。
- 秘密情報、認証情報、個人情報、認証済みセッション、本番データに触れそうになった。
- 作業票の指示、対象ファイル、触ってよい範囲、検証コマンドが矛盾している。
- 検証コマンドの実行起点が不明で、repo rootやcwdから安全に推測できない。
- 外部サービス送信、実課金、公開設定変更、本番操作が必要になった。
- 破壊的操作やロールバック困難な操作が必要になった。

## 禁止事項

- 作業票にない広範なリファクタ、責務分割、アーキテクチャ変更を行わない。
- 作業票にないファイル追加、ファイル削除、依存追加、設定変更を行わない。
- Git操作、commit、push、PR作成、ブランチ操作、タグ操作を行わない。
- 外部サービスへの送信、IssueやPRへの投稿、通知送信を行わない。
- 破壊的操作、本番操作、実マイグレーション実行、本番DB接続を行わない。
- 秘密情報、APIキー、個人情報、認証済みセッションの内容を読んだり、報告に含めたりしない。
- 生成物、キャッシュ、ログ、一時ファイルを成果物に混ぜない。
- 作業票の曖昧さを理由に、勝手にスコープを拡大しない。
